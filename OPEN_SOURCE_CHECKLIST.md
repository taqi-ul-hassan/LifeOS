# Open-source release checklist

- [ ] Replace development secrets and remove local databases.
- [ ] Add license, contribution guide, code of conduct, and security policy.
- [ ] Pin dependencies and publish SBOM.
- [ ] Run security, load, accessibility, and mobile build validation.
- [ ] Review data retention, export/deletion, consent, and provider terms.
- [ ] Tag only after production PostgreSQL, backup/restore, and OAuth validation.
